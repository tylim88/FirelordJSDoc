# DocumentId

This section discuss how `where` and field path sentinel `documentId()` types works.

import { Image } from '@site/src/components/Image'

## Is `__name__`

`documentId` return `__name__`, you can use `documentId()` and `__name__` interchangeably. The reason we keep `documentId` is because of backward compatibility.

```ts
// filter
where('__name__', '==', 'abc') // equivalent to where(documentId(),'==','abc')

// orderBy
orderBy('__name__') // equivalent to orderBy(documentId())
```

## Collection

When querying a collection by documentId(), you must provide a plain document ID, without any forward slash.
